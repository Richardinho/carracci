Unit: Core Model

The purpose of the Model is to represent the state of a component. The Model provides set and get methods for mutating its state. The Model also supports registering handler functions to be called whenever it changes state. These functions can be tied to specific properties or just any change event.

Handlers are called automatically on a change event, but these can be suppressed by passing the silent option to the set method. It is also possible to manually fire a change event
