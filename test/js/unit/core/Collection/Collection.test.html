Unit: Core Model

The purpose of the Model is to represent the state of a component. The Model provides set and get methods for mutating its state. The Model also supports the registering of handler functions to be called on state change. These functions can be tied to a general change event or to specific properties.

Handlers are called automatically by the Model, but this can be suppressed by passing a silent equals true option to the set method. It is also possible to manually fire a change event
